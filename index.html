O conceito

CSS em runtime,
sem surpresas.

O BreezeCSS observa o DOM com um MutationObserver e gera regras CSS a pedido, inserindo-as numa <style> gerida automaticamente. Sem pré-compilação.

⚡

Runtime puro

Nenhuma etapa de build. O motor lê as classes do HTML e gera o CSS correspondente directamente no browser, com cache inteligente para evitar re-processamento.

🔭

MutationObserver

Detecta automaticamente novos elementos adicionados ao DOM (via JS, frameworks, etc.) e aplica os estilos correspondentes sem que seja necessário chamar nada manualmente.

🎯

Valores arbitrários

Qualquer valor CSS é válido: p[23px], bg[#ff0], text-size[clamp(1rem,2vw,2rem)]. Sem lista pré-definida de valores.

🔌

Extensível via plugins

Adicione prefixos, variantes, breakpoints e processadores personalizados através de uma API clara de plugins, sem alterar o core da biblioteca.

Instalação

Dois métodos,
zero complexidade.

Escolha entre incluir o bundle directamente via <script> ou usar módulos ES para integrações mais avançadas.

Método 1 — Script directo Recomendado

Inclua o breeze.bundle.js no seu HTML. A biblioteca inicializa-se automaticamente quando o DOM estiver pronto.

index.html 
<!DOCTYPE html>
<html lang="pt">
<head>
  <meta charset="UTF-8">
  <title>A minha app</title>

  <!-- Incluir BreezeCSS bundle -->
  <script src="breeze.bundle.js"></script>
</head>
<body>

  <!-- As classes são aplicadas automaticamente -->
  <div class="flex items-center justify-between p[24px] bg[#111]">
    <h1 class="text-size[2rem] font-bold text[white]">Olá!</h1>
    <button class="px[20px] py[10px] bg[#00ff88] rounded-lg cursor-pointer hover:bg[#00cc6a] transition">
      Clicar
    </button>
  </div>

</body>
</html>
Auto-init: Ao incluir o bundle via <script>, a biblioteca inicializa-se automaticamente. Para desactivar, adicione o atributo data-no-auto-init à tag script.

Configuração inline (opcional)

Defina window.BreezeConfig antes do script para personalizar o tema, breakpoints e mais.

config inline 
<script>
  // Definir ANTES de carregar o bundle
  window.BreezeConfig = {
    debug: false,
    tema: {
      cores: {
        primario: '#00ff88',
        secundario: '#1a1a2e',
        accent: 'rgb(255,100,0)'
      },
      espacamento: {
        sm: '8px',
        md: '16px',
        lg: '32px'
      }
    },
    breakpoints: {
      tablet: '900px',  // breakpoint extra
    }
  };
</script>
<script src="breeze.bundle.js"></script>

Método 2 — Módulos ES

Para projetos que usam bundlers (Vite, Webpack, Rollup) ou queiram controlo total sobre a inicialização.

main.js — import básico 
// Importar a biblioteca (auto-init via DOM load)
import './breeze.js';
main.js — import com configuração avançada 
import BreezeCSS from './breeze.js';

// Desactivar auto-init e inicializar manualmente
BreezeCSS.init({
  debug: true,
  reset: true,
  tema: {
    cores: {
      primario: '#00ff88',
      bg: '#0a0d0f'
    },
    espacamento: {
      base: '16px',
      xl: '48px'
    }
  },
  breakpoints: {
    xxl: '1800px'  // adicionar ao topo dos existentes
  },
  plugins: [meuPlugin]
});

projeto/
├── breeze.js           # ponto de entrada
└── core/
    ├── engine.js       # motor principal
    ├── parser.js       # interpretação de classes

Sintaxe

Como escrever classes.

O BreezeCSS usa uma sintaxe expressiva com suporte a breakpoints, variantes e valores arbitrários.

anatomia de uma classe 
  [breakpoint:]  [variante:]  prefixo[valor]
       md:          hover:      bg[#00ff88]

  Exemplos:

  p[20px]                  → padding: 20px
  hover:bg[red]             → background-color: red on :hover
  md:text-size[1.5rem]     → font-size: 1.5rem at ≥768px
  md:hover:border[2px]    → border-width: 2px at ≥768px on :hover
  bg[to_right,red,blue]    → underscores viram espaços
  text[primario]           → resolve alias do tema
classes fixas
<div class="flex items-center gap[16px]">
  <span class="font-bold uppercase truncate">
    Título
  </span>
</div>
valores arbitrários
<div class="w[min(100%,480px)]
       bg[oklch(60%_0.2_240)]
       text-size[clamp(1rem,3vw,2rem)]">
  Conteúdo
</div>

Combine qualquer prefixo com variantes e breakpoints. A ordem é sempre: breakpoint:variante:classe.

exemplos combinados
<!-- Responsivo: esconder em mobile, mostrar em md -->
<div class="hidden md:block">Apenas desktop</div>

<!-- Hover com transição -->

API pública

Controlo total em runtime.

Todos os métodos disponíveis no objecto BreezeCSS (global) ou via import ES.

BreezeCSS.init() (config?: Object) → void

Inicializa a biblioteca com configuração opcional. Aceita tema, breakpoints, variantes, plugins, debug, reset e log.

BreezeCSS.processar() () → void

Processa manualmente todos os elementos do DOM actual. Útil após injecção dinâmica de HTML quando o MutationObserver não é suficiente.

BreezeCSS.build() (htmlString: string, opcoes?: { minificar: boolean }) → string

Modo build: recebe uma string HTML, processa todas as classes offline (sem tocar no DOM) e retorna o CSS gerado como string. Ideal para pipelines de build estático.

BreezeCSS.use() (plugin: { nome: string, setup: Function }) → void

Carrega e aplica um plugin. O plugin recebe um contexto ctx com acesso a addMapping, addFixedClass, addVariant, addBreakpoint e addProcessor.

BreezeCSS.addMapping() (prefixo: string, propriedade: string) → void

Adiciona em runtime um mapeamento prefixo → propriedade CSS. Ex: addMapping('my-prop', 'my-css-property').

BreezeCSS.addVariant() (nome: string, pseudo: string) → void

Adiciona uma variante personalizada. Ex: addVariant('group-hover', '.group:hover &').

BreezeCSS.addBreakpoint() (nome: string, tamanho: string) → void

Adiciona um breakpoint personalizado. Ex: addBreakpoint('tablet', '900px').

BreezeCSS.addComponent() (nome: string, classes: string[]) → void

Regista um componente como atalho para múltiplas classes. Ex: addComponent('btn', ['px[16px]', 'py[8px]', 'rounded-md']). Depois use class="btn".

BreezeCSS.addFixedClass() (nome: string, estilo: string) → void

Adiciona uma classe fixa (sem valor arbitrário) com estilos CSS directos. Ex: addFixedClass('glass', 'backdrop-filter: blur(10px)').

BreezeCSS.addProcessor() (prefixo: string, fn: (valor: string) => string) → void

Adiciona um processador personalizado que transforma o valor antes de aplicar. Ex: para blur[8px] → blur(8px).

BreezeCSS.stats() () → Object

Retorna estatísticas de cache (hits, misses, taxa de acerto) e informações da configuração actual (breakpoints, variantes, classes fixas, mapeamentos).

Extensão

Criar um plugin.

Um plugin é um objecto com um nome e uma função setup(ctx) que recebe o contexto da biblioteca.

exemplo de plugin completo
const meuPlugin = {
  nome: 'meu-plugin',

  setup(ctx) {

    // 1. Novo prefixo → propriedade CSS
    ctx.addMapping('filter2', 'filter');

    // 2. Classe fixa sem valor
    ctx.addFixedClass('glass',
      'backdrop-filter: blur(10px); background: rgba(255,255,255,0.05)'
    );

    // 3. Nova variante pseudo-classe
    ctx.addVariant('group-hover', '.group:hover &');

    // 4. Novo breakpoint
    ctx.addBreakpoint('xxl', '1800px');

    // 5. Processador de valor personalizado
    ctx.addProcessor('filter2', v => `blur(${v}) saturate(1.5)`);
  }
};

// Usar antes ou depois do init
BreezeCSS.use(meuPlugin);

// Ou passar no init:
BreezeCSS.init({ plugins: [meuPlugin] });

componentes como atalhos

Sistema de temas

Tokens de design centralizados.

Defina cores, espaçamentos e fontes no tema e use os seus aliases directamente nas classes.

definir tema
BreezeCSS.init({
  tema: {
    cores: {
      primario:   '#00ff88',
      secundario: '#1a1a2e',
      accent:     'rgb(255,100,0)',
      muted:      'rgba(255,255,255,0.4)'
    },
    espacamento: {
      xs:  '4px',
      sm:  '8px',
      md:  '16px',
      lg:  '32px',
      xl:  '64px'
    },
    fontes: {
      mono: 'Space Mono, monospace',
      sans: 'DM Sans, sans-serif'
    }
  }
});
usar aliases no HTML
<!-- Usar alias do tema directamente -->
<div class="bg[secundario] p[lg]">
  <h1 class="text[primario] font-family[mono]">
    Título
  </h1>
  <p class="text[muted] mt[sm]">
    Texto secundário
  </p>
  <button class="bg[primario] px[md] py[sm]
           hover:bg[accent] transition">
    CTA
  </button>
</div>

<!-- CSS variables geradas automaticamente -->
<!-- :root { --breeze-primario: #00ff88; ... } -->
CSS Variables: O BreezeCSS gera automaticamente variáveis CSS no :root para todos os tokens do tema (ex: --breeze-primario, --breeze-spacing-md, --breeze-font-mono), que podem ser usadas noutros estilos.
